## What is a Portfolio Webpage?

A portfolio webpage is a personal website that showcases who you are, what you've built, and how to reach you. It acts as a central place to present your skills, projects, and background to anyone — coordinators, collaborators, or recruiters.

## Technologies Used

The site was built using HTML and CSS only — no backend, no server required.

**HTML** handles the structure — every section, heading, card, and link on the page is defined in HTML.

**CSS** handles everything visual — colors, fonts, spacing, layout, and animations. CSS is essential here because HTML alone produces a plain unstyled text document. CSS is what turns that into an actual website with a dark theme, grid layouts, and smooth scroll behavior.

The layout was built using CSS Flexbox and Grid, which allow elements to arrange themselves cleanly across different screen sizes.

---

## How Responsiveness Was Achieved

Responsiveness means the website adapts its layout based on the screen size — it looks correct on both a phone and a laptop without breaking or requiring horizontal scrolling.

This was done using **CSS media queries**. A media query checks the width of the screen and applies different CSS rules depending on the result. For example, on desktop the project cards display in a multi-column grid. On mobile, the same cards stack vertically in a single column so they remain readable.

---

## Deployment

The site was deployed using **GitHub Pages** — a free static hosting service built into GitHub. The HTML and CSS files were pushed to a GitHub repository, and Pages was enabled from the repository settings. GitHub then automatically serves the files as a live public website.

No server setup, no hosting cost, no configuration needed beyond enabling the feature.
